# 公共契約

公共契約用於描述 MineAdmin 後端實現之間必須保持一致的部分。無論底層使用 Hyperf、Laravel，還是未來其他語言或框架，面向前台模板、插件和外部系統時都應遵守這些約定。

## 契約範圍

| 契約 | 説明 |
|------|------|
| [數據模型](./data-model.md) | 核心業務實體、關係和字段語義保持一致。 |
| [後台路由](./routing.md) | 後台資源、權限標識、操作語義和認證邊界保持一致。 |
| [接口元數據](./api-metadata.md) | OpenAPI/Swagger 元數據描述同一套請求和響應。 |
| [響應結構](./response.md) | API 返回結構、業務狀態碼、錯誤消息和分頁載荷保持一致。 |
| [前台模板對接](./frontend-template.md) | 前台模板通過穩定接口契約對接不同後端實現。 |

## 與框架實現的關係

公共契約只描述“必須一致”的行為，不綁定具體框架 API。框架實現文檔負責説明這些契約在對應框架中的落地方式，例如中間件註冊、服務容器、生命週期、事件系統、異常處理和文件上傳配置。

當前穩定實現是 [Hyperf 實現](/backend/frameworks/hyperf/)，Laravel 入口已在 [Laravel 實現](/backend/frameworks/laravel/1.0/) 中預留。
