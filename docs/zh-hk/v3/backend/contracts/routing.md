# 後台路由契約

後台路由契約定義管理端資源的訪問路徑、HTTP 方法、權限邊界和操作語義。不同框架的路由註冊方式可以不同，但對外暴露的後台接口應保持一致。

本文依據 MineAdmin 當前管理端控制器整理，描述前台模板、權限系統、接口文檔和審計日誌共同依賴的穩定約定。

## 基本原則

- 同一資源在不同實現中使用一致的路徑語義。
- 列表、詳情、創建、更新、刪除等常見操作保持穩定的 HTTP 方法和參數位置。
- 需要登錄的後台接口必須經過統一認證流程。
- 需要授權的後台接口必須綁定穩定的權限標識。
- 路由元數據應能支撐接口文檔、前台能力識別和操作日誌。
- 批量操作、導入導出、狀態切換等擴展操作應有清晰的資源歸屬。

## URL 命名空間

後台接口統一使用 `/admin` 作為管理端命名空間，命名空間後的第一段通常表示業務域或資源。

| 類型 | 路徑形態 | 示例 |
|------|----------|------|
| 認證接口 | `/admin/passport/{action}` | `/admin/passport/login`、`/admin/passport/refresh` |
| 當前用户能力 | `/admin/permission/{action}` | `/admin/permission/menus`、`/admin/permission/roles` |
| 資源管理 | `/admin/{resource}` | `/admin/user`、`/admin/role`、`/admin/department` |
| 資源列表 | `/admin/{resource}/list` | `/admin/user/list`、`/admin/attachment/list` |
| 資源子能力 | `/admin/{resource}/{id}/{relation}` | `/admin/role/{id}/permissions` |
| 插件後台接口 | `/admin/plugin/{plugin}/{action}` | `/admin/plugin/store/index` |

框架實現可以使用註解、路由表、控制器前綴或文件路由註冊這些地址，但最終公開 URL 不應因為框架差異而變化。

## 資源操作

| 操作 | HTTP 方法與路徑 | 參數位置 | 説明 |
|------|-----------------|----------|------|
| 列表 | `GET /admin/{resource}/list` | 查詢參數或請求參數 | 支持分頁、篩選、排序和數據權限過濾。 |
| 創建 | `POST /admin/{resource}` | JSON 請求體 | 請求體承載表單數據。 |
| 更新 | `PUT /admin/{resource}/{id}` | 路徑參數 + JSON 請求體 | 路徑參數定位資源，請求體承載可變更字段。 |
| 刪除 | `DELETE /admin/{resource}` | 請求參數或請求體 | 當前多數管理資源通過請求數據傳入單個或多個 ID。 |
| 單條刪除 | `DELETE /admin/{resource}/{id}` | 路徑參數 | 適用於已經明確用路徑參數定位的資源，例如附件。 |
| 關係讀取 | `GET /admin/{resource}/{id}/{relation}` | 路徑參數 | 查詢用户角色、角色權限等資源關係。 |
| 關係寫入 | `PUT /admin/{resource}/{id}/{relation}` | 路徑參數 + JSON 請求體 | 批量授予角色、權限或數據範圍。 |
| 資源動作 | `POST/PUT /admin/{resource}/{action}` | 依動作而定 | 上傳、重置密碼、狀態切換等非標準 CRUD 動作。 |

## 認證路由

認證路由屬於後台契約的一部分，但不按普通資源權限處理。

| 路徑 | 方法 | 認證要求 | 説明 |
|------|------|----------|------|
| `/admin/passport/login` | `POST` | 不需要訪問令牌 | 使用用户名和密碼換取訪問令牌與刷新令牌。 |
| `/admin/passport/logout` | `POST` | 需要訪問令牌 | 退出當前登錄狀態。 |
| `/admin/passport/getInfo` | `GET` | 需要訪問令牌 | 返回當前登錄用户的基礎信息和後台設置。 |
| `/admin/passport/refresh` | `POST` | 需要刷新令牌 | 換取新的訪問令牌。 |

## 當前用户能力路由

當前用户能力路由用於前台初始化，不等同於普通資源管理接口。

| 路徑 | 方法 | 説明 |
|------|------|------|
| `/admin/permission/menus` | `GET` | 返回當前用户可訪問的菜單樹。 |
| `/admin/permission/roles` | `GET` | 返回當前用户擁有的角色。 |
| `/admin/permission/update` | `POST` | 修改當前用户資料或密碼。 |

這些接口必須保持響應字段穩定，因為前台模板會用它們完成路由生成、菜單渲染和用户信息展示。

## 參數約定

- 分頁參數固定為 `page` 和 `page_size`，默認值分別為 `1` 和 `10`。
- 列表篩選字段隨資源定義，但應通過請求參數統一傳入，並由資源倉儲或服務解釋。
- 創建和更新接口使用 JSON 請求體，字段以對應表單請求或接口元數據為準。
- 文件上傳使用 `multipart/form-data`，附件上傳字段名固定為 `file`。
- 批量刪除接口通常使用 `ids` 或資源 ID 數組；如果接口已經聲明路徑參數，應以路徑參數定位單個資源。
- 用户授權角色使用 `role_codes` 數組，角色授權權限使用 `permissions` 數組，數組元素分別對應角色代碼和菜單權限標識。
- 崗位數據權限使用 `policy_type` 描述策略類型，`value` 描述策略值。

## 權限標識

權限標識應與後台菜單、按鈕和接口元數據保持一致。前台模板根據權限標識控制按鈕和頁面能力，後端實現根據同一標識執行授權校驗。

權限標識推薦使用 `業務域:資源:動作` 的形式，例如：

| 場景 | 權限標識示例 |
|------|--------------|
| 用户列表 | `permission:user:index` |
| 創建用户 | `permission:user:save` |
| 刪除用户 | `permission:user:delete` |
| 授權用户角色 | `permission:user:setRole` |
| 角色授權權限 | `permission:role:setMenu` |
| 附件上傳 | `dataCenter:attachment:upload` |
| 登錄日誌列表 | `log:userLogin:list` |

後端授權校驗應以該標識為準，前台菜單、按鈕權限和接口元數據也應複用同一標識，避免同一能力出現多個命名。

## 路由元數據

每個後台接口應提供穩定的路由元數據，至少覆蓋以下語義：

| 元數據 | 作用 |
|--------|------|
| `operationId` | 接口唯一操作名，用於接口文檔、客户端生成和變更識別。 |
| `summary` | 接口業務名稱，用於接口文檔和操作日誌。 |
| `tags` | 接口分組，應與後台功能模塊或資源歸屬一致。 |
| `security` | 聲明接口需要訪問令牌、刷新令牌或無需認證。 |
| 請求體 schema | 描述創建、更新、授權、上傳等接口的輸入字段。 |
| 響應 schema | 描述列表、詳情、當前用户能力等接口的返回結構。 |

如果某個框架沒有原生註解能力，也需要在路由註冊、OpenAPI 生成或等效配置中保存這些語義。

## 審計約定

需要記錄操作日誌的後台接口，應保留可識別的 HTTP 方法、真實路徑和業務名稱。操作日誌至少需要能夠追蹤：

- 操作用户。
- 請求方法。
- 請求路徑。
- 業務名稱。
- 客户端 IP。

因此，涉及新增、修改、刪除、授權、上傳等會改變系統狀態的接口，應確保路由元數據中的業務名稱明確，不使用空泛描述。

## 插件路由

插件可以註冊自己的後台路由前綴，但仍應遵守管理端路由契約：

- 路徑歸入 `/admin/plugin/{plugin}` 或插件明確聲明的後台命名空間。
- 需要登錄的插件接口必須使用統一認證流程。
- 會改變系統狀態的插件接口應使用合適的 HTTP 方法，並提供可審計的業務名稱。
- 插件接口返回結構應保持與後台統一響應契約一致。
