# 前台模板對接契約

前台模板對接契約用於保證同一套 MineAdmin 前台模板可以連接不同後端實現。只要後端實現遵守公共契約，前台不應因為底層框架不同而維護多套請求邏輯。

## 對接邊界

前台模板主要依賴以下穩定能力：

- 登錄、刷新 Token、退出登錄等認證接口。
- 用户信息、菜單、權限按鈕和角色能力。
- 後台資源的列表、詳情、創建、更新、刪除接口。
- 數據權限過濾後的業務數據。
- 文件上傳、附件訪問和導入導出接口。
- 統一響應結構和錯誤提示規則。

## 模板一致性

不同後端實現需要保持以下一致：

- 接口路徑和 HTTP 方法一致。
- 權限標識和菜單編碼一致。
- 請求參數和響應字段一致。
- 分頁、篩選、排序的參數語義一致。
- 文件上傳字段、返回附件結構和訪問地址語義一致。

## 框架差異處理

框架差異應收斂在後端實現層，例如路由註冊方式、中間件名稱、ORM 事件、異常處理器、隊列和文件系統配置。前台模板不應感知這些差異。

當某個框架實現無法完全複用公共契約時，需要在對應實現頁明確標註差異，並同步説明前台模板是否需要額外適配。
