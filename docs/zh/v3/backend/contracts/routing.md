# 后台路由契约

后台路由契约定义管理端资源的访问路径、HTTP 方法、权限边界和操作语义。不同框架的路由注册方式可以不同，但对外暴露的后台接口应保持一致。

## 基本原则

- 同一资源在不同实现中使用一致的路径语义。
- 列表、详情、创建、更新、删除等常见操作保持稳定的 HTTP 方法和参数位置。
- 需要登录的后台接口必须经过统一认证流程。
- 需要授权的后台接口必须绑定稳定的权限标识。
- 批量操作、导入导出、状态切换等扩展操作应有清晰的资源归属。

## 资源操作

| 操作 | 建议语义 | 说明 |
|------|----------|------|
| 列表 | 查询资源集合 | 支持分页、筛选、排序和数据权限过滤。 |
| 详情 | 查询单个资源 | 通过资源标识定位数据。 |
| 创建 | 新增资源 | 请求体承载表单数据。 |
| 更新 | 修改资源 | 请求体承载可变更字段。 |
| 删除 | 删除资源 | 可以是软删除或物理删除，由实现说明。 |
| 批量操作 | 对多个资源执行同一动作 | 需要明确权限和审计要求。 |

## 权限标识

权限标识应与后台菜单、按钮和接口元数据保持一致。前台模板根据权限标识控制按钮和页面能力，后端实现根据同一标识执行授权校验。

## 框架实现

Hyperf 当前通过注解、控制器和中间件承载路由与 API 文档生成，具体用法见 [Hyperf 路由与 API 文档](/v3/backend/frameworks/hyperf/base/router)。
