# 前臺模板對接契約

前臺模板對接契約用於保證同一套 MineAdmin 前臺模板可以連線不同後端實現。只要後端實現遵守公共契約，前臺不應因為底層框架不同而維護多套請求邏輯。

## 對接邊界

前臺模板主要依賴以下穩定能力：

- 登入、重新整理 Token、退出登入等認證介面。
- 使用者資訊、選單、許可權按鈕和角色能力。
- 後臺資源的列表、詳情、建立、更新、刪除介面。
- 資料許可權過濾後的業務資料。
- 檔案上傳、附件訪問和匯入匯出介面。
- 統一響應結構和錯誤提示規則。

## 模板一致性

不同後端實現需要保持以下一致：

- 介面路徑和 HTTP 方法一致。
- 許可權標識和選單編碼一致。
- 請求引數和響應欄位一致。
- 分頁、篩選、排序的引數語義一致。
- 檔案上傳欄位、返回附件結構和訪問地址語義一致。

## 框架差異處理

框架差異應收斂在後端實現層，例如路由註冊方式、中介軟體名稱、ORM 事件、異常處理器、佇列和檔案系統配置。前臺模板不應感知這些差異。

當某個框架實現無法完全複用公共契約時，需要在對應實現頁明確標註差異，並同步說明前臺模板是否需要額外適配。
