# バックエンドドキュメント

MineAdmin 3.x バックエンド開発ドキュメントへようこそ。このドキュメントは、MineAdmin のバックエンドアーキテクチャ、開発規範、ベストプラクティスを深く理解するのに役立ちます。

## アーキテクチャ概要

MineAdmin バックエンドは、最新の PHP テクノロジースタックを採用し、Hyperf 3.x フレームワークをベースに開発されており、高性能なコルーチン環境で動作します。

### コア機能

- **高性能コルーチン**: Swoole/Swow 拡張機能のコルーチン特性により、卓越した並行処理性能を提供
- **最新アーキテクチャ**: 明確な階層構造
- **安全・信頼性**: 組み込みのデュアルトークン認証メカニズム、完全な権限制御システム
- **拡張容易性**: プラグイン開発、モジュール化アーキテクチャ設計をサポート

## テクノロジースタック

| 技術コンポーネント | バージョン | 説明 |
|---------|------|------|
| PHP | 8.x+ | 最新 PHP 言語機能サポート |
| Hyperf | 3.x | 高性能コルーチンフレームワーク |
| Swoole/Swow | 最新安定版 | コルーチン実行環境 |
| MySQL | 5.7+ / 8.0+ | メインデータベース |
| Redis | 6.0+ | キャッシュとセッションストレージ |
| JWT | - | lcobucci/jwt ベースの認証スキーム |

## クイックスタート

### 環境要件

- PHP >= 8.1
- Swoole >= 5.1 または Swow >= 1.0
- MySQL
- Redis

### サービス起動

```bash
# HTTP サービスを起動
php bin/hyperf.php start
```

### 開発モード

```bash
# ホットリロードモード（開発環境推奨）
php bin/hyperf.php server:watch
```

## プロジェクト構成

MineAdmin のディレクトリ構成は Laravel の設計思想を参考にしており、Laravel 開発経験があればすぐに習得できます。

### ルートディレクトリ構成

```
├── App/              # アプリケーションコアコードディレクトリ
├── Config/           # 設定ファイルディレクトリ
├── Database/         # データベース関連ファイル
├── Storage/          # ストレージディレクトリ（ログ、アップロードファイルなど）
├── Tests/            # テストコードディレクトリ
├── Web/              # フロントエンドアプリケーションコード
└── Plugin/           # プラグインディレクトリ
```

### App ディレクトリ詳細

```
App/
├── Exceptions/       # 例外処理
├── Http/            # HTTP 関連（コントローラー、ミドルウェア、リクエストバリデーション）
├── Model/           # データモデル（Eloquent ORM）
├── Service/         # ビジネスロジック層
├── Repository/      # データアクセス層
└── Schema/          # API ドキュメントスキーマ定義
```

**アーキテクチャ階層の説明**:

- **Controller**: HTTP リクエスト、パラメータバリデーション、レスポンスフォーマットを処理
- **Service**: ビジネスロジックの編成、Repository と Model の調整
- **Repository**: データアクセスの抽象化、データソースの統一的提供（MySQL、Redis、ESなど）
- **Model**: データモデル定義、コルーチン版 Eloquent ORM ベース

## コア機能

### 認証認可

- **デュアルトークンメカニズム**: Access Token + Refresh Token によるシームレスな更新
- **RBAC 権限制御**: ロールベースのアクセス制御、データ権限をサポート
- **マルチエンドサポート**: Web、モバイル、API など複数のクライアントに対応

### データ権限

- **柔軟な設定**: ルールベースのデータ権限制御
- **多次元サポート**: 部門、ユーザー、ロールなど多次元の権限制御
- **透過的統合**: ビジネスロジックとシームレスに統合

### ログ監視

- **操作ログ**: ユーザー操作を詳細に記録
- **ログインログ**: ユーザーログイン記録とセキュリティ分析
- **システムログ**: アプリケーション実行ログとエラートレース
