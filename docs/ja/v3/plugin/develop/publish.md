# アプリケーションの公開

アプリケーションをパッケージ化し、アプリマーケットに公開して、他のユーザーがダウンロードして使用できるようにします。

## アプリケーションのパッケージ化

現在、プラグインアプリケーションのディレクトリ全体を git リポジトリプロジェクトとして扱う方法を提供しています。`github`、`gitee` などのコードホスティングプラットフォームでホストできます。
また、MineAdmin が自社で構築した GIT サーバー http://git.mineadmin.com でホストすることもできます。

### パッケージ化の手順

1. アプリケーションディレクトリを git リポジトリプロジェクトとして、コードホスティングプラットフォームにコミットします。

```shell
cd あなたのプラグインアプリケーションディレクトリ

git init

git add .

git commit -m "first commit"

git remote add origin あなたのコードリポジトリのアドレス

git push -u origin master
```

2. [MineAdmin プラグイン作成ページ](https://www.mineadmin.com/member/createApp) にアクセスし、コードリポジトリのアドレスを入力して送信します。

<ElImage :preview-src-list="['/images/create_app.png']" src="/images/create_app.png"></ElImage>

3. MineAdmin の審査が通過すると、アプリケーションがアプリマーケットに表示されます。

::: warning パッケージ化に関する注意事項

* `mine.json` の必須項目を必ず全て記入してください。記入がないとアプリケーションが正常に公開されません。
* アプリケーションアップロード後、弊社で審査を行います。審査通過後、アプリマーケットに表示されますので、ご了承ください。
* プラグインディレクトリに `install.lock` ファイルが含まれていないことを確認してください。含まれていると、アプリケーションが正常にインストールできません。

:::

## アプリケーションのバージョン管理

`MineAdmin` エコシステムの健全性、信頼性、安全性を維持するために、所有するアプリケーションに重要なアップデートを行う際は、以下の基準に従うことを推奨します。
<a href="https://semver.org/lang/zh-CN/" target="_blank">セマンティック バージョニング仕様</a>
に基づいて新しいバージョンを公開してください。

### 推奨事項
アプリケーションのバージョンは 1.0.0 から始め、以下のように増分することを推奨します。

| コードの状態                | 段階の説明             | ルール            | バージョン例 |
|:-----------------------|:------------------|:---------------|:--------|
| コードの初回リリース            | 新バージョンリリース        | 1.0.0 から開始     | 1.0.0   |
| コードの後方互換性があるバグ修正      | バグ修正、パッチバージョンリリース | 3桁目の数字を増やすことを推奨 | 1.0.1   |
| コードの後方互換性がある新機能追加     | 小機能追加、マイナーバージョンリリース | 2桁目の数字を増やすことを推奨 | 1.1.0   |
| コードの破壊的で後方互換性がない変更    | 破壊的更新、メジャーバージョンリリース | 1桁目の数字を増やすことを推奨 | 2.0.0   |
