# MineAdmin 国際化設定完全ガイド

MineAdmin フロントエンドは Vue i18n v11 をベースに構築された完全な国際化ソリューションを提供し、多言語切り替え、動的言語パック読み込み、モジュール式翻訳管理をサポートします。このドキュメントでは、システムの国際化実装メカニズムと使用方法について詳しく説明します。

## 概要

### サポートされている言語
- 簡体字中国語 (zh_CN) - デフォルト言語
- 繁体字中国語 (zh_TW)
- 英語 (en)

### コア機能
- 🌐 多言語動的切り替え
- 📦 モジュール式言語パック管理
- ⚡ 動的言語パック読み込み
- 🔧 開発ツール統合
- 📱 グローバルおよびローカルスコープのサポート
- 🎯 TypeScript 型安全性

### 技術スタック
- **Vue i18n**: 11.1.2
- **ビルドプラグイン**: @intlify/unplugin-vue-i18n 6.0.3
- **言語形式**: YAML (JSON サポート)
- **型システム**: TypeScript

## プロジェクト設定

### システム初期化

システムは `/web/src/bootstrap.ts` で国際化設定を初期化します。

```typescript
// ソースコード位置: web/src/bootstrap.ts:44-67
// GitHub: https://github.com/mineadmin/mineadmin/blob/master/web/src/bootstrap.ts

async function createI18nService(app: App) {
  // 言語パックファイルを自動スキャン
  const locales: any[] = Object.entries(import.meta.glob('./locales/*.y(a)?ml')).map(([key]: any) => {
    const [, value, label] = key.match(/^.\/locales\/(\w+)\[([^[\]]+)\]\.yaml$/)
    return { label, value }
  })
  
  useUserStore().setLocales(locales)
  
  // メッセージ形式を処理
  Object.keys(messages as any).map((name: string) => {
    const matchValue = name.match(/(\w+)/) as RegExpMatchArray | null
    if (messages && matchValue) {
      messages[matchValue[1]] = messages[name]
      delete messages[name]
    }
  })

  // i18n インスタンスを作成
  app.use(createI18n({
    legacy: false,                    // Composition API を使用
    globalInjection: true,            // グローバル注入
    fallbackLocale: 'zh_CN',         // フォールバック言語
    locale: useUserStore().getLanguage(), // 現在の言語
    silentTranslationWarn: true,      // 翻訳警告を非表示
    silentFallbackWarn: true,         // フォールバック警告を非表示
    messages,                         // 言語パックデータ
  }))
}
```

## 言語パック管理

### ファイル命名規則

言語パックファイルは特定の形式に従う必要があります：`言語識別子[自言語名].yaml`

```bash
# 正しい命名形式
zh_CN[简体中文].yaml
en[English].yaml  
zh_TW[繁體中文].yaml

# 誤った命名形式
zh_CN.yaml         # 言語名が不足
en[英文].yaml      # 言語名が誤り
zh-CN[简体中文].yaml  # 言語識別子の形式が誤り
```

### グローバル言語パック

システムは以下の3つの場所の言語パックを自動スキャンします。

1. **コア言語パック**: `src/locales/`
2. **モジュール言語パック**: `src/modules/<モジュール名>/locales/`
3. **プラグイン言語パック**: `src/plugins/<プラグイン名>/locales/`

::: danger 重要なお知らせ
**モジュール** と **プラグイン** 配下の言語パックファイル名は `src/locales` 配下のファイル名と完全に一致している必要があります。一致しない場合、コンソールに翻訳キーが見つからないという警告が表示されます。
:::

**コア言語パック例** (`src/locales/zh_CN[简体中文].yaml`)：

## Hook システム詳細

### useTrans Hook

**ソースコード位置**: `web/src/hooks/auto-imports/useTrans.ts`  
**GitHub**: https://github.com/mineadmin/mineadmin/blob/master/web/src/hooks/auto-imports/useTrans.ts

```typescript
import { useI18n } from 'vue-i18n'
import type { ComposerTranslation } from 'vue-i18n'

export interface TransType {
  globalTrans: ComposerTranslation
  localTrans: ComposerTranslation
}

export function useTrans(key: any | null = null): TransType | string | any {
  const global = useI18n()
  const local = useI18n({
    inheritLocale: true,
    useScope: 'local',
  })

  if (key === null) {
    return {
      localTrans: local.t,
      globalTrans: global.t,
    }
  }
  else {
    // グローバル翻訳を優先し、次にローカル翻訳を検索
    return global.te(key) ? global.t(key) : local.te(key) ? local.t(key) : key
  }
}
```

### useLocalTrans Hook

**ソースコード位置**: `web/src/hooks/useLocalTrans.ts`  
**GitHub**: https://github.com/mineadmin/mineadmin/blob/master/web/src/hooks/useLocalTrans.ts

```typescript
import { useI18n } from 'vue-i18n'
import type { ComposerTranslation } from 'vue-i18n'

export function useLocalTrans(key: any | null = null): string | ComposerTranslation | any {
  const { t } = useI18n({
    inheritLocale: true,
    useScope: 'local',
  })
  return key === null ? t as ComposerTranslation : t(key) as string
}
```

## パフォーマンス最適化

### 遅延読み込み言語パック

```typescript
// 必要な時に言語パックを読み込む
const loadLanguage = async (locale: string) => {
  try {
    const messages = await import(`../locales/${locale}[${getLocaleName(locale)}].yaml`)
    const { global } = useI18n()
    global.setLocaleMessage(locale, messages.default)
    global.locale.value = locale
  } catch (error) {
    console.error(`言語パックの読み込みに失敗しました: ${locale}`, error)
  }
}
```

### キャッシュ最適化

```typescript
// 翻訳結果をキャッシュ
const translationCache = new Map<string, string>()

const cachedT = (key: string, ...args: any[]) => {
  const cacheKey = `${key}-${JSON.stringify(args)}`
  if (translationCache.has(cacheKey)) {
    return translationCache.get(cacheKey)
  }
  
  const result = t(key, ...args)
  translationCache.set(cacheKey, result)
  return result
}
```

## デバッグとトラブルシューティング

### よくある問題

#### 1. 翻訳キーが見つからない

**問題**: コンソールに警告 `Not found 'xxx' key in 'zh_CN' locale messages.` が表示される。

**解決策**:
```typescript
// キーが存在するか確認
const { te, t } = useI18n()
const safeT = (key: string, fallback: string = key) => {
  return te(key) ? t(key) : fallback
}
```

#### 2. モジュール言語パックが反映されない

**問題**: モジュールまたはプラグインの言語パックが読み込まれない

**解決策**:
1. ファイル名の形式が正しいことを確認：`zh_CN[简体中文].yaml`
2. ファイルパスがスキャン範囲内にあることを確認
3. YAML の構文が正しいことを確認

#### 3. 型エラー

**問題**: TypeScript の型チェックに失敗する

**解決策**:
```typescript
// 正しい型定義を使用
import type { TransType } from '@/hooks/auto-imports/useTrans.ts'

const i18n = useTrans() as TransType
const t = i18n.globalTrans  // 型安全な翻訳関数
```

### カスタム翻訳ロジック

```typescript
// カスタム翻訳 Hook を作成
export function useCustomTrans() {
  const { t, te } = useI18n()
  
  return {
    t: (key: string, fallback?: string) => {
      if (te(key)) {
        return t(key)
      }
      
      // カスタムフォールバックロジック
      if (fallback) {
        return fallback
      }
      
      // 欠落した翻訳キーを記録
      console.warn(`欠落した翻訳: ${key}`)
      return key
    }
  }
}
```
