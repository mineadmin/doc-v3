# リリースノート

## バージョン管理方式

MineAdmin は `>=3.0` 以降、[セマンティック バージョニング](https://semver.org/) を採用しています。
メジャーバージョンは年に1回、マイナーパッチは `毎週または毎月` に1回リリースされます。

<el-alert type="warning">
つまり、アプリケーションでは <el-tag type="danger">~3.0</el-tag> のようなバージョン制約を可能な限り採用すべきです。
</el-alert>

## 名前付き引数

[名前付き引数](https://www.php.net/manual/en/functions.arguments.php#functions.named-arguments) は現在のところ全面的には採用されていません。
必要に応じて `クラス名`、`クラスメソッド名`、`関数名` を変更する場合があります。これは MineAdmin のコードを改善するためです。

現在、公式が採用している命名規則は以下のとおりです。

1. すべてのクラスは `アッパーキャメルケース` を使用します。
2. すべてのクラスメソッドは `ローワーキャメルケース` を使用します。
3. 関数は `スネークケース` を使用します。

## サポート計画

すべての MineAdmin バージョンにおいて、`バグ修正` は18ヶ月間、`セキュリティ修正` は2年間提供されます。
